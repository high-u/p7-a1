# 会話システム仕様書

## 1. 概要

VLM を脳とするAIエージェントが、複数ターンの会話を通じて一貫した対話を継続するための仕組み。

## 2. 設計方針

### セッションなし、スライディングウィンドウ

セッションという概念を持たない。常に最新 N 件のメッセージが「現在の意識」として存在し続ける。新しいメッセージが追加されれば、最古のものが窓から外れる。

AIは眠り続けることができる。1ヶ月前の会話の続きでも、N件未満であれば自然に継続する。セッション間の橋渡しは記憶システム（別仕様）が担う。

### VLM を脳とする一貫性

脳（VLM）に渡すのは常に画像。テキスト変換を挟まない。

画像の検索・インデックス化はシステムの責務であり、脳の関心事ではない。同じ VLM を画像説明文の生成に使うことがあるが、それは処理の一部であり、脳としての役割ではない。

### ツール履歴を体験として記録

会話履歴にはテキストのやり取りだけでなく、ツール呼び出しと結果も含める。AIが「何を言ったか」だけでなく「何をしたか」が記録として残ることで、連続した体験が生まれる。

## 3. VLM と画像

| | 役割 | 画像の扱い |
|---|---|---|
| 脳（VLM）| リアルタイム知覚・応答 | 常に画像として受け取る |
| システム処理 | 検索用テキスト生成 | 同じ VLM を使用。脳としてではなく処理の一部として |

システムが生成した画像説明文は検索専用。脳は見ない。

## 4. 会話ログの性質

### 二つの役割

1. **現在のコンテキスト**: スライディングウィンドウとして、エージェントのマルチターン会話を実現する
2. **能動的な想起**: `search_conversation` ツールでAIが過去の会話を検索できる

### 保存対象

| role | 内容 |
|---|---|
| `user` | ユーザーのプロンプト |
| `assistant` | エージェントの応答テキスト、またはツール呼び出し |
| `tool` | ツールの実行結果 |

## 5. ツール

| ツール | 概要 |
|---|---|
| `search_conversation` | 過去の会話ログをキーワードで検索する |

> 検索の引数・ロジック・レスポンス形式の詳細は未決定。
